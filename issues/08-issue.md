# Визуальный редизайн чата: карточки вместо рамок, markdown-lite

**Контекст.** Ядро UX готово — теперь внешний вид. Убираем «коробочность»,
делаем воздух и акценты. Это чисто `ui/`-задача, поведение не трогаем.

**Шаги:**

1. **Главный layout:** убрать `Borders::ALL` вокруг чата. Новый вид:
   ```text
    filar ▸ user@host:22          ⠹ thinking      always
   ──────────────────────────────────────────────────────
     you                                                  ← accent, bold
     Проверь свободное место на диске
                                                          ← пустая строка
     agent                                                ← success, bold
     Проверяю диски…
                                                          
     ▾ $ Get-PSDrive -PSProvider FileSystem  ✓            ← warning
     │ Name  Used(GB)  Free(GB)                           ← fg_dim, gutter │
     │ C     201.5     36.2
                                                          
   ──────────────────────────────────────────────────────
    ❯ введите сообщение…                                  ← prompt-строка
   ──────────────────────────────────────────────────────
    enter send   ! shell   ^T terminal   ^P password   ^C quit
   ```
   - Статус-бар: без сплошной заливки `DarkGray` во всю ширину. Слева
     `filar ▸ {target}` (accent — только имя таргета), по центру/справа —
     индикатор режима словом и цветом (Normal — ничего не показывать,
     Thinking — спиннер+`warning`, Confirm — `danger`), справа —
     `confirm_mode` тускло. Разделитель под баром — строка `─` цветом
     `fg_muted`.
   - Роль-заголовки: строчные `you` / `agent` без символов-префиксов,
     bold + цвет роли. Тело — обычный `fg` с отступом 1 пробел.
   - Блок команды: заголовок из задачи 6; строки вывода — с gutter `│ `
     цветом `fg_muted` вместо `  | `.
   - `System` — одна строка `· text` (`fg_muted`), `Error` — `✗ text` / `! text`
     (danger).
   - Help-бар: без заливки фоном; клавиши — `fg_dim`, описания — `fg_muted`,
     разделение тремя пробелами. Клик по элементу help-бара выполняет действие
     (hit-test по прямоугольникам слов: `^T` → toggle interactive, `^C` → quit
     и т.д.) — заполнить `app.helpbar_zones: Vec<(Rect, HelpAction)>` при рендере.
2. **Поле ввода:** без рамки. Строка-разделитель сверху и снизу, слева промпт
   `❯ ` (accent; ASCII fallback `> `). Плейсхолдер `введите сообщение…` —
   `fg_muted`. При вводе `!` в начале — промпт меняется на `$ ` и цвет ввода
   на `warning` (мгновенная обратная связь «это shell-команда»). Многострочный
   рост: до 5 строк (`Constraint::Length` вычислять от wrap ввода), дальше —
   внутренний скролл к курсору. PasswordInput — промпт `🔒 `/`* `, маска как
   сейчас.
3. **Markdown-lite для сообщений `Agent`** (только инлайн, без полного парсера):
   - \`code span\` → стиль: `fg` на фоне `surface` (или accent без фона, если
     фон выглядит грязно в conhost);
   - `**bold**` → `Modifier::BOLD`;
   - fenced-блоки ``` ``` ``` → строки с gutter `│ ` и цветом `fg_dim`,
     первая строка — язык тусклым;
   - заголовки `#`/`##` в начале строки → bold accent, символы `#` убрать;
   - маркеры списков `- ` → `• `.
   Реализовать в `ui/text.rs` как `fn render_markdown_line(&str, theme) -> Vec<Span>`
   + распознавание fence-состояния на уровне блока. Не поддерживаем: таблицы,
   ссылки, вложенность — сознательно, ради лёгкости.
4. Проверить всё в Windows Terminal и conhost; для conhost предусмотреть
   ASCII-фоллбэки (`❯`→`>`, `·`→`-`, `│`→`|`, `✓`→`[ok]`). Определение: env
   `WT_SESSION` есть → современный терминал; иначе — консервативный набор.
   Вынести в `theme.rs` структуру `Glyphs { prompt: &str, gutter: &str, ... }`.

**DoD:**
- Ни одной сплошной рамки в основном экране (кроме модала подтверждения).
- Скриншот-проверка: чат читается «по ролям» без напряжения, цвета не кричат.
- Markdown-lite: код-спаны, bold, fence-блоки, заголовки, списки рендерятся.
- В conhost всё отображается без «?» (ASCII-фоллбэки работают).
- Unit-тесты `render_markdown_line`: code span, bold, смешанный случай,
  незакрытый маркер (рендерится как обычный текст).

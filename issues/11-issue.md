# Полировка, устойчивость, документация

**Шаги:**

1. **Стабильность layout:** пройтись по всем режимам — высоты зон фиксированы
   или меняются плавно (рост поля ввода — единственное легитимное изменение).
   Убрать `terminal.clear()` на каждое agent-событие (`needs_clear`), если после
   редизайна артефактов нет — полный clear вызывает мерцание; оставить только
   на смену режима.
2. **Ресайз:** сжать окно до 40×10 — ничего не паникует (все `saturating_sub`
   на месте, модал клампится, кэш перестраивается).
3. **Деградация:** запуск в терминале без поддержки мыши — приложение работает
   как раньше (ошибки включения mouse capture игнорировать с `warn!`).
4. **Help-бар** обновить под новые возможности (упомянуть мышь: `wheel scroll`,
   `click expand`, `drag copy` — коротко, в 1 строку, лишнее убрать).
5. **Тесты:** прогнать полный набор `cargo test --workspace`; добавить
   недостающие unit-тесты (hit_test, SSE-парсер, markdown-lite, кламп скролла).
6. **Документация:** обновить `README.md` (Features: Mouse support, Streaming
   responses; новый скриншот-плейсхолдер) и `USER_GUIDE.md` (раздел «Мышь»:
   скролл, клики, выделение/копирование; изменение — Enter в диалоге
   подтверждения активирует выделенную кнопку, дефолт Deny).
7. `CHANGELOG`/`PROGRESS.md` — записать итог, поднять версию до `0.2.0`
   в `workspace.package`.

**DoD:**
- `cargo build --release`, `cargo clippy --all-targets -- -D warnings`,
  `cargo test --workspace` — зелёные.
- Ручной smoke-тест сценария: запуск → вопрос агенту → стриминг ответа →
  подтверждение команды кликом → разворот вывода кликом → выделение и копия
  текста → Ctrl+T → скролл колесом → Ctrl+T назад → Ctrl+C → сессия сохранена.
- Документация отражает новые возможности.

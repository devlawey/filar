# Сворачиваемые блоки команд + клики по ним

**Контекст.** Вывод команды обрезается на 30 строках навсегда. Делаем
expand/collapse: по умолчанию показываем 5 строк, остальное — по клику.

**Шаги:**

1. В `App`: `pub collapsed_overrides: HashMap<usize, bool>` (индекс блока →
   пользователь явно свернул/развернул). Дефолт: блок `Command` с выводом
   длиннее 6 строк — свёрнут до 5 строк + строка-переключатель.
2. В `layout_cache` при построении: для свёрнутого блока после 5 строк вывода
   добавлять строку `▸ … 42 more lines — click to expand` (`fg_muted`, регион
   `OutputToggle`); для развёрнутого длинного — после вывода строку
   `▾ collapse` (`fg_muted`). Прежний жёсткий лимит 30 строк убрать (кэш
   выдерживает; для развёрнутых блоков оставить страховочный потолок 400 строк
   с пометкой `… truncated`).
3. `handle_mouse`: клик по строке региона `OutputToggle` → инвертировать
   состояние в `collapsed_overrides`, инкрементировать `message_rev`
   (перестройка кэша). Клик по строке-заголовку `Command` — тоже toggle
   (удобная большая цель для клика).
4. Заголовок блока команды сделать компактнее и информативнее:
   `▸ $ docker ps -a   ✓` (свёрнут) / `▾ $ docker ps -a   ✓` (развёрнут);
   `✓` — success, `✗` — danger для denied; если терминал conhost — ASCII
   fallback `[ok]/[no]` (использовать существующий подход `strip_emoji`,
   символы `▸▾✓✗` входят в whitelist-диапазоны — проверить, при необходимости
   добавить диапазон 0x2713–0x2717).

**DoD:**
- Длинный вывод по умолчанию свёрнут до 5 строк, клик разворачивает/сворачивает.
- Состояние сохраняется при скролле и новых сообщениях (индексы не съезжают).
- Заголовок команды показывает статус и стрелку-индикатор.

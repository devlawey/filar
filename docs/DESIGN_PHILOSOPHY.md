# Filar TUI — Дизайн-философия

> **Этот документ читается перед началом КАЖДОЙ задачи из плана модернизации
> (`TUI_MODERNIZATION_PLAN.md`) и применяется ко всем принимаемым решениям.**
> Если конкретный шаг задачи противоречит этим принципам — принципы важнее,
> отклонение зафиксировать комментарием в коде и в PROGRESS.md.

---

## Цель

Современный, лёгкий, лаконичный TUI в духе **Warp / Claude Code / lazygit**.
Мышь — полноправный инструмент наравне с клавиатурой. Никакой перегрузки,
никаких украшательств. Стильно за счёт сдержанности, а не за счёт эффектов.

Мысленный тест для любого нового элемента интерфейса:
**«Убирает ли это трение для пользователя?»** Если нет — элемент не нужен.

---

## Принципы

### 1. Минимум рамок

Сплошные `Borders::ALL` вокруг каждой зоны — устаревший «коробочный» вид.
Вместо рамок:

- **воздух** — пустые строки между блоками;
- **тонкие горизонтальные разделители** (`─`) между крупными зонами;
- **вертикальная акцент-полоса / gutter** (`│`) слева от вложенного контента
  (вывод команд, код-блоки).

Единственное легитимное место для рамки — **модальный диалог** (подтверждение
команды): там рамка отделяет модал от контента под ним. Углы скруглённые
(`BorderType::Rounded`).

### 2. Единая тема

Все цвета — **только** из структуры `Theme` (`crates/tui/src/ui/theme.rs`).
Ни одного литерала `Color::*` вне theme.rs. Палитра приглушённая:

| Роль | Токен | Назначение |
|---|---|---|
| Акцент | `accent` | пользователь, фокус, промпт — один цвет на всё приложение |
| Успех | `success` | agent, approved-команды |
| Внимание | `warning` | thinking, команды, shell-escape |
| Опасность | `danger` | ошибки, деструктивные команды, deny |
| Текст | `fg` / `fg_dim` / `fg_muted` | три градации: основной / вторичный / подсказки |
| Поверхность | `surface` | фон кнопок и «приподнятых» элементов |

Один акцентный цвет. Три градации серого. Больше цветов не добавлять.

### 3. Цвет = смысл

Цвет несёт информацию: роль сообщения, статус команды, режим приложения.
Если элемент не несёт смысла — он серый (`fg_dim`/`fg_muted`) или его нет вовсе.
Никогда не красить «для красоты». Проверка: если убрать цвет с элемента и
интерфейс не потерял информацию — цвет был лишним.

### 4. Ничего не мигает и не прыгает

- Единственная анимация в приложении — **спиннер** во время работы агента.
- Layout стабилен: зоны не меняют размер и позицию при смене режима
  (единственное легитимное изменение — плавный рост многострочного поля ввода).
- Никаких полных `terminal.clear()` без необходимости — это мерцание.
- Никаких эффектов появления, «печатающих машинок» с искусственной задержкой,
  градиентов, теней.

### 5. Мышь дополняет клавиатуру, не заменяет

- Каждое мышиное действие имеет клавиатурный эквивалент.
- Хоткеи в help-баре остаются всегда видимы.
- Приложение полностью управляемо без мыши (терминалы без mouse-capture —
  деградация без потери функций, только с потерей удобства).

### 6. Обратная совместимость

Все существующие механики работают как раньше и не ломаются ни одной задачей:

- хоткеи, **включая поддержку русской раскладки ЙЦУКЕН** (`ctrl_key(en, ru)`);
- shell-escape `!команда`;
- Ctrl+T (интерактивный терминал), Ctrl+P (ввод пароля), Ctrl+C (выход);
- секреты `$FILAR_SECRET_N`, session persistence, hot-swap транспорта.

Единственное согласованное изменение поведения: **Enter в диалоге подтверждения
активирует выделенную кнопку (дефолт — Deny)**, а не безусловный approve.
Это безопаснее и отражается в help-баре.

### 7. Windows-first, деградация — предусмотрена

Проект целится в Windows: проверка **обязательно** в Windows Terminal *и*
в conhost. Для conhost — ASCII-фоллбэки глифов через структуру `Glyphs`
в theme.rs (детект по env `WT_SESSION`):

| Современный | Fallback |
|---|---|
| `❯` | `>` |
| `│` | `\|` |
| `·` | `-` |
| `✓` / `✗` | `[ok]` / `[no]` |
| `▸` / `▾` | `+` / `-` |
| braille-спиннер `⠋⠙⠹…` | `\|/-\` |

Ни один символ не должен отображаться как `?`.

### 8. Производительность — часть дизайна

«Лёгкий» — значит и по ресурсам: рендер не выше 60 fps, layout чата кэшируется
и перестраивается только при инвалидации (ширина, новые сообщения, ревизия
последнего блока), спиннер тикает только в режиме Thinking. CPU в простое — ноль.

## Anti-scope: чего НЕ делаем

Сознательные ограничения — не «пока не успели», а решение:

- ❌ вкладки, панели файлового менеджера, деревья, виджеты CPU/RAM;
- ❌ выбор тем в UI (одна выверенная тёмная тема; светлая — отдельная итерация);
- ❌ полный markdown (таблицы, картинки, ссылки, вложенность) — только
  markdown-lite: code-спаны, bold, fenced-блоки, заголовки, маркеры списков;
- ❌ градиенты, тени, анимации появления, эффект печатающейся машинки;
- ❌ command palette и автодополнение — кандидаты на следующую итерацию,
  не на эту.

Если в ходе задачи возникает соблазн добавить что-то из этого списка —
не добавлять.

## Целевой вид (референс)

```text
 filar ▸ user@host:22          ⠹ thinking      always
──────────────────────────────────────────────────────
  you
  Проверь свободное место на диске

  agent
  Проверяю диски…

  ▾ $ Get-PSDrive -PSProvider FileSystem  ✓
  │ Name  Used(GB)  Free(GB)
  │ C     201.5     36.2

──────────────────────────────────────────────────────
 ❯ введите сообщение…
──────────────────────────────────────────────────────
 enter send   ! shell   ^T terminal   ^P password   ^C quit
```

Читается «по ролям» без напряжения. Цвета не кричат. Ничего лишнего.

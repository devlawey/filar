# Filar — Пользовательская инструкция

> Терминал с AI-агентом поверх SSH. Агент выполняет команды на локальной или
> удалённой машине, запрашивая подтверждение перед каждой операцией.

---

## 1. Быстрый старт

### 1.1. Запуск (самый простой способ)

1. Откройте Проводник и перейдите в `c:\dev\filar`
2. Дважды кликните файл **`run.bat`**
3. Откроется GUI-окно — введите модель, URL и API-ключ прямо в интерфейсе
4. Нажмите **Launch**

### 1.2. Запуск через PowerShell

```powershell
cd c:\dev\filar
.\target\debug\filar.exe
```

Откроется GUI-лаунчер. Без аргументов — всегда GUI.

### 1.3. CLI-режим (напрямую в TUI, без GUI)

```powershell
# Локальная машина (требуется GLM_API_KEY в env)
$env:GLM_API_KEY = "ваш-ключ"
.\target\debug\filar.exe --target local

# SSH-таргет + конкретный LLM-профиль
.\target\debug\filar.exe --target test-docker --llm glm

# Восстановить предыдущую сессию
.\target\debug\filar.exe --target local --session 1718900000
```

### 1.4. Сборка из исходников

```powershell
$env:PATH = "$env:USERPROFILE\.cargo\bin;$env:USERPROFILE\mingw\mingw64\bin;$env:PATH"
cd c:\dev\filar
cargo build
```

Бинарник: `target\debug\filar.exe`

## 2. Конфигурация (config.toml)

### 2.1. Структура

```toml
# Режим подтверждения команд: always | allowlist | never
confirm_mode = "always"

# LLM по умолчанию (профиль "default")
[llm]
model = "glm-5.1"
api_base_url = "https://open.bigmodel.cn/api/paas/v4"
max_tokens = 4096

# Дополнительные LLM-профили (опционально)
[[llm_profiles]]
name = "deepseek"
model = "deepseek-chat"
api_base_url = "https://api.deepseek.com/v1"
max_tokens = 8192
key_env = "DEEPSEEK_API_KEY"   # имя env-переменной с ключом

# SSH-таргеты
[[ssh_targets]]
name = "my-server"
host = "192.168.1.100"
port = 22
user = "admin"

[ssh_targets.auth]
type = "key"              # key | password | agent
path = "~/.ssh/id_ed25519"  # только для type = "key"

[[ssh_targets]]
name = "test-docker"
host = "127.0.0.1"
port = 2222
user = "testuser"

[ssh_targets.auth]
type = "password"         # пароль из env SSH_PASSWORD

# Таймауты (в секундах)
[timeouts]
command_secs = 120
llm_secs = 60
connect_secs = 15
```

### 2.2. Режимы подтверждения

| Режим       | Поведение                                                         |
|-------------|-------------------------------------------------------------------|
| `always`    | Каждая команда требует явного approve/deny (по умолчанию, безопасно) |
| `allowlist` | Read-only команды (ls, cat, ps...) выполняются автоматически; остальные — с подтверждением |
| `never`     | Все команды выполняются без подтверждения (опасно!)               |

### 2.3. API-ключи

**Основной способ (GUI):** ввести ключ прямо в интерфейсе запуска.

**Альтернатива (CLI-режим):** через переменные окружения:

```powershell
# Для LLM по умолчанию (GLM)
$env:GLM_API_KEY = "ваш-ключ"

# Для дополнительного профиля (например, DeepSeek)
$env:DEEPSEEK_API_KEY = "ваш-ключ-deepseek"

# Пароль для SSH (если auth type = "password")
$env:SSH_PASSWORD = "пароль-ssh"
```

Ключи **не хранятся** в config.toml.

## 3. GUI-лаунчер

При запуске без аргументов открывается окно "Filar — Launcher":

```text
┌──────────────────────────────────────────┐
│  Filar                                    │
│  Terminal with an AI agent over SSH       │
│──────────────────────────────────────────│
│  Recent sessions:                         │
│    + Start new session                    │
│    2025-06-20 14:30 | local | Помог…      │
│    2025-06-19 10:15 | my-server | …       │
│──────────────────────────────────────────│
│  Target:              [local        ▼]   │
│  LLM profile (preset): [default    ▼]    │
│  Model:               [glm-5.1        ]   │
│  API base URL:        [https://open.big… ]│
│  API key:             [··············]    │
│──────────────────────────────────────────│
│          [Launch]  [Cancel]              │
└──────────────────────────────────────────┘
```

- **Recent sessions** — список до 10 последних сессий (клик восстанавливает)
- **Target** — `local` или SSH-таргет из конфига
- **LLM profile (preset)** — пресет из конфига; при выборе автозаполняет поля Model и URL
- **Model** — имя модели (можно править вручную)
- **API base URL** — адрес API (можно править вручную)
- **API key** — ваш ключ к LLM API (вводится прямо в интерфейсе)
- **Launch** — запускает TUI с выбранными параметрами

## 4. TUI — Режим агента

Основной режим работы. Вы пишете задачу на естественном языке, агент
выполняет команды и возвращает результат.

### 4.1. Горячие клавиши

| Клавиша         | Действие                                    |
|-----------------|---------------------------------------------|
| `Enter`         | Отправить сообщение / Подтвердить выбранную кнопку |
| `Ctrl+C`        | Выход из приложения                         |
| `Ctrl+T`        | Переключение между режимом агента и терминалом |
| `Ctrl+P`        | Ввод пароля (маскированный ввод)            |
| `Up/Down`       | Навигация по истории ввода                  |
| `!command`      | Shell escape (прямой вызов команды)         |
| `a` / `y`       | Подтвердить команду (в режиме подтверждения)|
| `d` / `n`       | Отклонить команду (в режиме подтверждения)  |
| `Tab`           | Переключение между кнопками в диалоге (Deny → Approve) |

### 4.2. Управление мышью

| Действие        | Результат                                  |
|-----------------|--------------------------------------------|
| Колесо вверх/вниз | Прокрутка истории чата (по 3 строки)    |
| Клик по `▾`/`▸` | Развернуть/свернуть вывод команды          |
| Клик по заголовку команды | Развернуть/свернуть вывод        |
| Клик по `↓ N new` | Прокрутка к последним сообщениям         |
| Drag (зажать ЛКМ и тянуть) | Выделение текста и копирование в буфер |
| Двойной клик    | Выделение слова                            |
| Тройной клик    | Выделение строки                           |

В режиме подтверждения команды:
- Клик по кнопке **Approve** / **Deny** — выбирает действие
- `Enter` активирует выделенную кнопку (по умолчанию — **Deny**)

В интерактивном терминале (Ctrl+T):
- Колесо — скролл истории терминала (или стрелки в less/man)
- Клик/перетаскивание — проброс мыши в приложение (vim, htop, mc)

### 4.3. Режимы приложения

- **Normal** (поле ввода) — ввод текста, отправка агенту
- **Thinking** (агент работает) — ожидание ответа LLM / выполнения команд
- **Confirming** (подтверждение) — агент просит разрешение на команду
- **Interactive** (терминал) — обычный эмулятор терминала

### 4.4. Блоки чата

Каждое сообщение отображается цветным блоком:

| Цвет     | Тип       | Описание                              |
|----------|-----------|---------------------------------------|
| Cyan     | User      | Ваше сообщение                        |
| Green    | Agent     | Ответ/рассуждение агента              |
| Yellow   | Command   | Команда + вывод + статус (✓/✗)       |
| Red      | Error     | Сообщение об ошибке                   |
| Gray     | System    | Системное уведомление                 |

### 4.5. Подтверждение команд

Когда агент предлагает выполнить команду, вы видите:

```
⚠ COMMAND CONFIRMATION REQUIRED
Agent wants to run:  sudo apt update
Explanation:         Updating package lists to check available versions
⚠ WARNING: This command may modify system state

[a]pprove  /  [d]eny
```

- Нажмите `a` или `y` — команда выполняется
- Нажмите `d` или `n` — команда отклоняется, агент продолжит работу
- `Ctrl+C` — отклонить и выйти

## 5. TUI — Интерактивный терминал

Нажмите `Ctrl+T` в режиме агента для переключения в обычный терминал.

```
┌──────────────────────────────────────────┐
│ INTERACTIVE                    local     │ ← статус-бар
│──────────────────────────────────────────│
│ user@machine:~$ ls                       │
│ Documents  Downloads  Music  Pictures    │
│ user@machine:~$ _                        │ ← курсор
│──────────────────────────────────────────│
│ Ctrl+T=Agent mode │ (terminal input is forwarded) │ ← help-бар
└──────────────────────────────────────────┘
```

- Работает локальный PTY (команды выполняются на вашей машине)
- Полная поддержка ANSI: `vim`, `htop`, `less`, `top` и т.д.
- Ресайз окна автоматически меняет размер терминала
- `Ctrl+T` — вернуться в режим агента
- При выходе из шелла (exit/EOF) — автоматический возврат в режим агента

## 6. Сессии

### 6.1. Автосохранение

Каждая сессия **автоматически сохраняется** при выходе из TUI (Ctrl+C или
закрытие терминала).

Сохраняется:
- ID сессии (Unix timestamp)
- Timestamp (дата и время)
- Имя таргета (local / SSH-таргет)
- Имя LLM-профиля
- Все сообщения чата (включая вывод команд)

### 6.2. Хранилище

- **Windows:** `%APPDATA%\filar\sessions\`
- **Unix:** `~/.filar/sessions/`

Формат: JSON-файлы (`<id>.json`). Хранится не более **10 последних** сессий
(старые удаляются автоматически).

### 6.3. Восстановление

**Через GUI:** кликнуть на сессию в списке "Recent sessions" → Launch.

**Через CLI:**
```powershell
.\target\release\filar.exe --target local --session 1718900000
```

При восстановлении загружается вся история чата. В начале добавляется
системное сообщение: "Session restored — history loaded from disk".

## 7. Логи

Пока TUI активен, логи **не пишутся в терминал** (иначе строки лога легли бы
поверх интерфейса). Вместо этого весь лог идёт в файл, а сообщения уровня
**WARN и ERROR дополнительно дублируются в чат** как системные строки
(например, `filar_transport::ssh: reader: channel closed` при разрыве SSH).

### 7.1. Где лежит файл

- **Windows:** `%APPDATA%\filar\logs\filar.log`
- **Unix:** `~/.filar/logs/filar.log`

Файл ротируется посуточно (`filar.log.YYYY-MM-DD`). В нём — полная запись
с временными метками, включая уровни ниже WARN.

### 7.2. Уровень логирования

Уровень берётся из переменной окружения `RUST_LOG` (по умолчанию `info`).
Чтобы поднять детализацию:

```powershell
$env:RUST_LOG = "debug"
.\target\release\filar.exe
```

Можно фильтровать по модулю, например только транспорт:
`$env:RUST_LOG = "filar_transport=debug,info"`.

## 8. CLI-аргументы

```
Usage: filar [--target <name>] [--llm <profile>] [--session <id>]

Options:
  --target <name>   Подключиться к таргету ('local' или имя SSH-таргета из конфига)
  --llm <profile>   Использовать LLM-профиль ('default' или имя из [[llm_profiles]])
  --session <id>    Восстановить сессию по ID
  -h, --help        Показать справку
```

Без аргументов → GUI-лаунчер.

### Примеры

```powershell
# Локальная работа, LLM по умолчанию
filar --target local

# SSH на сервер, DeepSeek
filar --target my-server --llm deepseek

# Восстановить сессию на локальной машине
filar --target local --session 1718900000

# Только GUI (без аргументов)
filar
```

## 9. Типичный сценарий работы

### Сценарий 1: Диагностика через агента

1. Запустите: `filar --target my-server`
2. Введите: "Найди, какой процесс слушает порт 8080, и покажи его рабочую директорию"
3. Агент предложит: `sudo lsof -i :8080` → подтвердите (a)
4. Агент предложит: `ls -la /proc/<pid>/cwd` → подтвердите (a)
5. Агент выдаст итоговый ответ с путём

### Сценарий 2: Быстрый фикс через терминал

1. Запустите: `filar --target local`
2. Нажмите `Ctrl+T` — переключитесь в терминал
3. Выполните нужные команды вручную
4. Нажмите `Ctrl+T` — вернитесь к агенту
5. Спросите агента: "Проверь, что сервис nginx запущен"

### Сценарий 3: Восстановление вчерашней сессии

1. Запустите без аргументов: `filar`
2. В GUI выберите вчерашнюю сессию из списка
3. Нажмите Launch — история загрузится
4. Продолжите диалог: "А теперь проверь логи за сегодня"

## 10. Решение проблем

### "failed to load config"
- Проверьте, что `config.toml` существует в рабочей директории
- Или укажите путь: `$env:FILAR_CONFIG = "C:\path\to\config.toml"`

### "API key is required"
- В GUI-режиме: введите ключ в поле "API key" и нажмите Launch
- В CLI-режиме: установите переменную `$env:GLM_API_KEY = "ваш-ключ"`

### "SSH target 'xxx' not found"
- Проверьте имя таргета в `config.toml` (поле `name` в `[[ssh_targets]]`)
- Список доступных таргетов смотрите в `config.toml` в секциях `[[ssh_targets]]`

### SSH-подключение не удаётся
- **Пароль:** установите `$env:SSH_PASSWORD`
- **Ключ:** проверьте путь в `config.toml` → `[ssh_targets.auth]` → `path`
- **Порт:** проверьте поле `port` (по умолчанию 22)

### TUI отображается некорректно
- Убедитесь, что терминал поддерживает ANSI (Windows Terminal, PowerShell 7)
- Минимальный размер окна: ~80×24

### Сессии не сохраняются
- Проверьте права на запись в `%APPDATA%\filar\sessions\`
- Создайте директорию вручную при необходимости

## 11. Безопасность

- **Ключи API** вводятся в GUI или через env-переменные, никогда не хранятся в config.toml
- **Режим `always`** (по умолчанию): ни одна команда не выполняется без вашего
  подтверждения
- **Деструктивные команды** (rm -rf, mkfs, dd...) помечаются предупреждением ⚠
- **Zero-install:** на удалённой машине не создаётся никаких файлов —
  команды выполняются в существующем шелле через SSH-канал
